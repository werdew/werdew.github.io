Git reference

Staging: 

		git add -A           # stages all changes (equivalent: --all)
		git add .            # stages all changes in current folder and subfolders.
		git add example.txt  # stage file 'example.txt'
		git add *            # stages all matching files (e.g. '*.txt') (no deletions, shell-expanded, may miss hidden)
		

		git status
		git reset
Commit: confirming and saving changes permanently. 

		git commit -m "comment"
		git reset HEAD~
		git rm
		git reset
		git reset --hard
		git rm -f four.txt
		git rm --cached four.txt
		git rm -r myFolder
View commits: 

		git log
		git log --oneline
Branching: 

		git branch
		git branch development
		git checkout development
		git checkout main
		git checkout development
		git merge main -m "merging main to development"
		git checkout main
		git merge development -m "Merging on main with development"
Merging: Conflict: 

		git log --oneline
Previous Version: 

		git log --oneline
		git checkout 4153dca #precious version
Compare: 

		git diff 'newest_id' 'oldest_id'
Push: sending local changes to the remote 

		git push origin main
		git push origin staging
		git push origin development
Fetch: Bringing remote changes into your local repository, but not merging 

		git fetch
		git merge
Pull: Fetching plus merging 

		git pull
Restore: 

		git restore one.txt
		git restore folder
		git restore .
		git restore --staged .    #restore staged
Stash: for switching branches, stash your current branch 

		git stash
		git stash pop  #restores stash
		git stash apply
		git stash list
		git stash pop stash@{0}
		git stash apply stash@{0}
		git stash drop
Revert: 

		git revert b80e820
Reset: Rebase: 

		git rebase main
Pull Request: [PR] Mac Setting: 

		# Globally excluding .DS_Store file, which is used for mac UI
		git config --global core.excludesfile ~/.gitignore_global
		echo .DS_Store >> ~/.gitignore_global
